## 2026-05-04 — Fix: extension activated on creatio.com marketing site

**What changed:** Tightened page-type detection in `src/pageDetection.js`:
- Shell detectors reduced to platform-specific custom elements only (`crt-app-toolbar`, `crt-root`, `crt-page`, `crt-schema-outlet`, `crt-reusable-schema`, `mainshell`, `ShellContainerWithBackground`, `[data-item-marker="AppToolbarGlobalSearch"]`). Removed the false-positive signals: `script[src*="creatio"]`, `link[href*="creatio"]`, `[class*="shell"]`, `[id*="shell"]`, `[class*="Terrasoft"]`, `[id*="Terrasoft"]`, `script[src*="terrasoft"]`, `link[href*="terrasoft"]`, `.creatio-logo`. `minRequired` lowered to 1 since each remaining selector is a strong, platform-only signal.
- Login detectors reduced to strong markers: `#loginEdit-el`, `#passwordEdit-el`, `.login-button-login`, and exact path match for `/login`/`/auth`. Removed broad selectors `[class*="login"]`, `[id*="login"]`, `currentUrl.includes("login")`, generic `input[name*="username|password|login"]`, `form[action*="login"]`, title/meta heuristics.
- Added marketing subdomains to `EXCLUDED_DOMAINS` (`src/menuConfig.js` and the mirrored list in `login/login.js`): `www.creatio.com`, `marketplace.creatio.com`, `partners.creatio.com`, `events.creatio.com`, `blog.creatio.com`.
- Added 5 unit tests in `tests/pageDetection.test.js` covering creatio.com marketing root, excluded subdomains, and false-positive rejection.

**Why:** On `creatio.com` (the marketing portal) the floating "Clio satellite" menu was rendered. Root cause: the marketing site loads its own JS/CSS, so `script[src*="creatio"]` and `link[href*="creatio"]` matched the shell heuristic; the "Log in" CTA also matched `[class*="login"]`/`[id*="login"]`. The plugin must only run on the actual Creatio platform (Shell/Configuration UI), not on company-owned marketing/portal sites.

**Decision:** Combined approach — primary fix is removing weak DOM signals (works on every domain, including unknown future ones); secondary defense-in-depth is an explicit subdomain blocklist for the well-known marketing properties. Bare `creatio.com` was deliberately not added to the blocklist, because the current `hostname.includes(domain)` matcher would also exclude legitimate `*.creatio.com` platform tenants. The tightened detection alone correctly returns `null` for the marketing root (verified by the new test).
